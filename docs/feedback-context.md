# Feedback Context

Kazane records external feedback as Context before turning it into implementation work.
This preserves why the person raised the issue instead of reducing the report to a
detached feature request.

From **Context Cards**, select **Feedbackを記録** and enter:

1. where the feedback was received;
2. the trial user or organization, which may be anonymous;
3. the usage background and desired outcome;
4. the observation, difficulty, or proposal;
5. confirmation that consent to record it for product improvement was obtained.

The resulting card is marked `Feedback`, stores the consent record and minimal
participant information, and keeps the observation as an unresolved point. Select
that unresolved point to create a linked Work Item without losing the original
background.

Do not enter unnecessary personal information. External sharing remains a human
decision and is not implied by consent to local recording.
