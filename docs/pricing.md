## 将来の価格設計（案）

以下はドラフトであり、コミットメントではありません。GA リリース時に確定します。

### 設計原則

1. **ローカルファーストの価値に見合う価格**: データはユーザーのマシンに留まる。クラウド SaaS の従量課金とは異なるモデルを検討。
2. **AI モデルコストの分離**: Kazane 自体は AI モデルを含まない。Claude・OpenAI 等のコストはユーザーが個別に負担。
3. **小規模組織に手の届く価格**: 初期ターゲット（1〜10名チーム）が試せる範囲。

### 検討中のモデル

| モデル | 概要 | 検討優先度 |
|--------|------|-----------|
| 個人ライセンス（買い切り） | 1シート・1マシン | 高 |
| チームライセンス（年額） | 複数シート・管理ポータル | 中（リモート GUI 実装後） |
| オープンソース + サポート | コアをOSS化・エンタープライズサポート有償 | 検討中 |
| SaaS（ホスティング型） | クラウド版 Kazane | 低（v0.9以降の課題） |

### 未確定の前提条件

- リモート GUI（ADR-0014）の実装範囲と時期
- 複数ユーザー管理の UI 実装
- Apple Developer Program への参加（署名・App Store 配布）
- サポート体制の整備
