# v0.9 Local Candidate Rehearsal

Date: 2026-07-23  
Candidate: Kazane 0.9.0 arm64, unsigned local candidate

## Result

The local candidate path passes on the current macOS user. This is implementation
evidence for WI-904, WI-905, and WI-907, but it does not close the clean-user,
Developer ID signing, notarization, external trial, or release decision gates.

## Verified

- `scripts/build-candidate --no-sign` produced both `Kazane.app` and
  `Kazane_0.9.0_aarch64.dmg`.
- `hdiutil verify` reported a valid DMG checksum.
- The app was ad-hoc signed for local execution, installed at
  `/Applications/Kazane.app`, and reported version 0.9.0.
- The installed app opened the existing local database without data loss.
- Data Location created an integrity-checked SQLite backup from the UI.
- Data Location exported diagnostics from the UI without repository knowledge.
- The generated backup returned `ok` from `PRAGMA integrity_check`.
- The diagnostics JSON contained no email address, name, record body, content,
  description, local absolute path, or database path field.
- Frontend build, 16 Python tests, Rust check/tests, and dependency audit passed.

## Preserved rollback material

- The pre-upgrade database was preserved beside the local Kazane data.
- The prior application bundle was preserved under the local `app-backups`
  directory before replacement.

Absolute, user-specific backup paths are intentionally excluded from this report.
